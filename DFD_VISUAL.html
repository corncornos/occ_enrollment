OCC Enrollment System

Comprehensive Data Flow Diagrams (DFD) - Current Implementation

📋 Implementation Notes:

  • Enhanced Enrollment Workflow: System automatically classifies students as regular/irregular and routes through appropriate approval chain
  • Enrollment Approval Flow: Regular: Student → Registrar → Admin → Confirmed. Irregular: Student → Program Head → Registrar → Admin → Confirmed

  • Adjustment Period Management: New process (9.0) for post-enrollment adjustments with workflow: Student → Program Head → Dean → Registrar Staff → Approved
  • All 9 Major Processes: Fully implemented including adjustment period management
  • All 21 Data Stores: Complete implementation including adjustment period control and changes tracking
  • Registrar Staff Role: Handles enrollment assistance, COR generation, and adjustment period final approval
  • Dean Role: Reviews curriculum submissions and adjustment period requests

Context Diagram (Level 0)

Overview: The Context Diagram represents the highest level view of the OCC Enrollment System, showing the system as a single process and its interactions with external entities.

External Entities:

  • Student - Primary user who registers, applies for enrollment, uploads documents, views schedules and grades
  • Admin/Registrar - System administrator who manages users, curriculum, sections, schedules, approves enrollments, generates CORs, and manages grades
  • Admission Officer - Handles applicant processing, document verification, student number assignment, and enrollment control
  • Program Head - Submits curriculum proposals, reviews irregular student enrollments, and reviews adjustment period requests for their respective programs
  • Dean - Reviews and approves curriculum submissions, reviews adjustment period requests, and views enrollment reports
  • Registrar Staff - Handles enrollment assistance, generates CORs, and approves adjustment period requests

Level 0 DFD

Overview: The Level 0 DFD decomposes the OCC Enrollment System into major processes, showing how data flows between processes, external entities, and data stores.

Enhanced Features:

  • Enrollment Processing (4.0) includes automatic student classification (regular/irregular) and multi-stage approval workflow: Regular (Student → Registrar → Admin), Irregular (Student → Program Head → Registrar → Admin)
  • COR Generation happens at Registrar stage before final admin approval
  • Adjustment Period Management (9.0) - New process for post-enrollment adjustments with workflow: Student → Program Head → Dean → Registrar Staff
  • Program Head participates in enrollment review for irregular students and adjustment period approvals
  • Registrar Staff handles enrollment assistance, COR generation, and final adjustment period approval
  • Dean reviews curriculum submissions and adjustment period requests

Major Processes:

1.0 User Management - Handles user registration, authentication, and account management
2.0 Application Processing - Manages student applications, document verification, and applicant workflow
3.0 Curriculum Management - Handles curriculum submissions, reviews, and approvals
4.0 Enrollment Processing - Manages enrollment requests with regular/irregular workflows, section assignments, schedule creation, and COR generation
5.0 Grade Management - Handles grade entry, verification, and viewing
6.0 Document Management - Manages document uploads, verification, and status updates
7.0 Reporting - Generates enrollment reports and analytics
8.0 Chatbot Service - Processes queries and provides FAQ responses
9.0 Adjustment Period Management - Manages post-enrollment adjustments with multi-stage approval workflow

4.0 Enrollment Processing (Level 1)

Enhanced Workflow: The system automatically classifies students as regular or irregular based on grades.

  • Regular Students: Student → Registrar (generates COR) → Admin → Confirmed
  • Irregular Students: Student → Program Head → Registrar (generates COR) → Admin → Confirmed

Status Flow: draft → pending_program_head (irregular) / pending_registrar (regular) → pending_admin → confirmed

COR Generation: Registrar generates COR after reviewing enrollment, before forwarding to Admin for final approval
